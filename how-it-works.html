How the World Wide Web Works

The Web is built on three core technologies invented by Tim Berners-Lee: HTML (the content), HTTP (the transport), and URLs (the addressing). Together, they form a global hypertext system where any document can link to any other.

The Three Pillars of the Web

📝

HTML — HyperText Markup Language

HTML is the language used to write web pages. It uses tags to mark up content — headings, paragraphs, images, tables, and crucially, hyperlinks (<a href="...">) that connect pages.

Current version: HTML5 (2014)

🔄

HTTP — HyperText Transfer Protocol

HTTP is the communication protocol that governs how browsers request documents from servers. A request ("GET this page") and a response ("200 OK — here is the HTML") form the basis of every web interaction.

Current version: HTTP/3 (2022)

📌

URL — Uniform Resource Locator

A URL is a unique address for every resource on the Web. It specifies the protocol (https://), the domain, and the path to the specific document — enabling any page to link to any other page.

Standard: RFC 3986

Hyperlinks in HTML

A hyperlink is the fundamental unit of the Web. It is created using the HTML anchor tag <a>, which wraps around any piece of content — text, images, buttons — and turns it into a clickable link.

The href (Hypertext REFerence) attribute specifies the destination: it can be an absolute URL (pointing to any site on the web), a relative URL (pointing to another page in the same website), an anchor to a section on the same page, or even a mailto: link to open an email client.

Types of Hyperlinks

TypeExampleUse Case
Absolute https://w3.org Linking to external websites
Relative about.html Linking between pages in the same site
Anchor #section-id Jumping to a section on the same page
Email mailto:info@example.com Opening the user's email client

Web Design Principles in Practice

Good websites apply established design principles to make information clear, consistent, and usable.

Principle Definition Example on This Site
Structure Organise information into a clear, logical hierarchy Header → Nav → Main content → Footer; semantic HTML5 tags
Simplicity Include only what is necessary; reduce clutter Clean layout, limited colour palette, no decorative distractions
Visibility Make important elements easy to see and find Large headings, contrasting buttons, clear navigation labels
Feedback Confirm user actions with visual responses Hover effects on menu items and cards; active link highlighting
Tolerance Allow users to navigate without getting stuck Navigation is always visible; every page links back to all others
Reuse Use consistent components and styles throughout Single external CSS file; same header, nav and footer on all pages

The Impact of Hypertext

📚

Education

Online learning platforms, Wikipedia, academic journals, and MOOCs have made knowledge freely accessible to billions of people worldwide.

🛒

Commerce

E-commerce generates trillions in revenue annually. Hyperlinks are the pathways through which consumers discover, evaluate, and purchase products.

🤝

Communication

Social media, email, messaging platforms, and video calls are all built on the Web's infrastructure of linked, addressable resources.
